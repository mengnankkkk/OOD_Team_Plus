# Money Whisperer：多 Agent 目标理财管家设计

## 1. 项目概述

Money Whisperer 是一款面向理财小白和普通上班族的 AI 原生理财顾问产品。用户只需描述自己的生活目标、财务情况与担忧，系统便可调度一组专业 Agent，完成数据获取、财务诊断、市场研究、资产配置、风险验证和通俗解释。

产品前台强调简单、可信和可行动，后台强调真实金融任务、多 Agent 协作、工具调用、确定性计算、风控否决和完整审计。

比赛原型采用以下边界：

- 读取模拟账户、手工录入数据或 CSV，不连接真实交易。
- 允许生成增配、减配、持有和等待建议，但只支持模拟采纳。
- 不承诺收益，不用单一行情信号直接生成买卖结论。
- 所有建议必须绑定用户目标、风险等级、证据、有效期和失效条件。

## 2. 双赛道契合

### 2.1 AI 原生理财产品赛道

- 产品定位是投顾和决策陪伴工具，而非行情、K 线或资讯聚合软件。
- 基于真实用户财务状态、风险偏好和人生阶段生成个性化建议。
- 主动识别追涨杀跌、集中持仓和目标偏离等问题。
- 使用普通用户能理解的语言解释建议、风险和不确定性。
- 提供持续跟踪，而不是一次性问卷和静态配置模板。

### 2.2 金融 Agent 实战赛道

- Agent 能完成数据获取、研究、配置、风险分析和报告生成等复杂任务。
- 多个 Agent 按工作流并行或串行协作，不是简单角色接龙。
- Agent 通过工具访问数据和确定性金融引擎。
- 工作流包含任务规划、状态管理、冲突处理、失败重试和结果复核。
- 系统能展示完整执行轨迹、工具调用记录和反馈闭环。

## 3. 目标用户

### 3.1 核心人群

- 22 至 40 岁、收入相对稳定的上班族。
- 有一定存款或基金、股票等资产，但缺乏系统理财知识。
- 正在为买房、应急金、教育、养老等目标积累资金。
- 资产分散在银行卡、基金平台和证券账户中。
- 容易受到短期行情和社交媒体观点影响。

### 3.2 核心痛点

1. 不知道自己真正能承担多少投资风险。
2. 不知道现金、稳健资产和权益资产应该如何分配。
3. 看不懂专业投研结论，也无法判断建议是否可靠。
4. 市场波动时容易追涨杀跌。
5. 账户和目标相互割裂，缺少全局视角。
6. 传统聊天机器人只能回答问题，不能完成完整决策任务。

## 4. 产品原则

1. **目标优先**：先判断用户要解决什么生活问题，再讨论资产和产品。
2. **组合优先**：买卖判断必须基于整个资产组合，而不是孤立标的。
3. **风险优先**：先保障应急金、流动性和风险边界，再讨论收益。
4. **证据优先**：建议必须展示数据、逻辑、假设和不确定性。
5. **行动可逆**：默认使用分批、模拟和观察方案，避免激进的一次性操作。
6. **人机共决**：Agent 提出方案，用户保留最终决策权。
7. **规则兜底**：LLM 不得绕过确定性计算、适当性规则和合规审查。

## 4.1 官方资源带来的设计增强

官方资源不是简单的“数据接口列表”，而是可以组合成一条可复现的投研验证链：

- **QuantSkills**：把数据路由、因子分析、风险监控、回测和 Agent 研究方法封装成可加载的 Skill/Agent 资产。
- **PandaAI AI 工作流**：提供自然语言生成策略、模板模式、空白模式、AI 对话模式、可视化节点连线、参数调整和自动化回测。
- **PandaAI 超级图表**：用于把因子、策略和条件触发位置可视化，帮助用户理解“建议来自什么证据”。
- **PandaAI 仿真模块**：提供模拟信号、模拟持仓、模拟盈亏和运行日志，适合作为比赛演示的验证沙盒，但不能被包装成真实交易或投资建议。
- **Pandadata API**：覆盖交易日历、股票、基金、指数、期货、期权、港美股、宏观和因子等数据域。

产品因此新增一个后台概念：**Evidence Lab（证据实验室）**。普通用户只看到小白友好的建议卡；评委或进阶用户可以展开 Evidence Lab，查看数据源、Skill、工作流节点、计算结果、验证报告和风险拦截原因。

## 5. 功能需求

### 5.1 P0：比赛原型必须实现

#### 5.1.1 对话式建档

- 采集收入、固定支出、可变支出、负债、家庭责任和现有资产。
- 采集目标金额、目标时间和目标优先级。
- 通过情景问题判断风险承受能力，而不是只使用静态问卷。
- 检测“主观偏好激进但实际承受能力低”等回答矛盾。
- 明确展示数据用途、授权范围和删除入口。

#### 5.1.2 只读资产聚合

- 支持模拟账户、手工录入和 CSV 导入。
- 将现金、存款、货币基金、债券基金、权益基金和股票统一归类。
- 识别跨账户重复持仓和隐含行业集中度。
- 为每笔数据保留来源、时间戳和新鲜度状态。

#### 5.1.3 财务健康诊断

- 计算结余率、负债率、应急金覆盖月数和资产集中度。
- 判断短期目标资金是否暴露在过高波动中。
- 输出最重要的三项问题，不用大量指标淹没用户。
- 每项问题给出“为什么重要”和“建议先做什么”。

#### 5.1.4 目标规划

- 支持买房、应急金、教育、养老和自定义目标。
- 计算资金缺口、建议月投入、可接受风险和预计达成概率。
- 支持多个目标之间的优先级冲突分析。
- 提供稳健、均衡、进取三档方案，并解释取舍。

#### 5.1.5 AI 顾问

- 用户可询问“市场下跌要不要卖”“这笔奖金如何分配”等问题。
- Agent 可主动追问缺失信息。
- 回答先给结论摘要，再按需展开证据和专业细节。
- 每次回答都显示数据时间、置信度和风险提示。
- 用户可要求比较其他方案、查看反方观点或重新计算。

#### 5.1.6 自选管理

- 支持添加基金、股票、指数和其他观察资产。
- 用户可填写关注理由、计划持有期限及关联目标。
- 自选项显示估值状态、风险变化、组合关联和最新 Agent 结论。
- 支持价格、回撤、估值、重大事件和组合占比提醒。
- 普通价格波动默认不推送，避免鼓励频繁交易。

#### 5.1.7 回撤提醒

- 支持用户设置 10%、15%、20% 或自定义回撤阈值。
- 回撤应基于明确的阶段高点和时间窗口计算。
- 触发阈值后，风险 Agent 先判断市场、行业、标的和组合层面的原因。
- 提醒分为信息、关注、重要和紧急四级。
- 提醒必须说明回撤幅度、影响目标、Agent 判断和下一步建议。

#### 5.1.8 买入建议

买入建议使用“增配”表达，并至少包含：

- 建议标的或资产类别。
- 与用户目标和配置缺口的关系。
- 建议投入金额或组合占比区间。
- 一次投入或分批投入的节奏。
- 数据依据、主要风险和替代方案。
- 建议有效期与失效条件。
- 模拟采纳后的组合风险和目标成功率变化。

买入建议不能仅由价格下跌、单一技术指标或热点事件触发。

#### 5.1.9 卖出建议

卖出建议使用“减配”或“退出观察”表达，主要触发条件包括：

- 单一资产或行业集中度超过用户约束。
- 用户目标期限缩短，流动性需求上升。
- 风险特征发生明显变化。
- 投资逻辑失效或关键数据恶化。
- 组合再平衡需要。

卖出建议至少包含建议比例、执行节奏、税费或成本提示、替代配置、风险变化和失效条件。

#### 5.1.10 模拟采纳

- 用户点击建议后先进入模拟页面，不直接执行。
- 展示调仓前后资产结构、预期波动、历史回撤和目标成功率变化。
- 支持比较“采纳、不操作、采用替代方案”三种结果。
- 用户可记录接受、拒绝或稍后决定，并填写原因。
- 用户反馈进入后续建议的偏好学习流程。

#### 5.1.11 提醒中心

- 统一展示回撤、目标偏离、配置失衡和买卖观察信号。
- 支持仅重要提醒、每日汇总和完全静默等频率设置。
- 相似提醒应合并，避免重复推送。
- 每项提醒必须提供查看分析、模拟方案和忽略三个动作。

#### 5.1.12 决策日志

- 记录建议生成时间、数据版本、Agent 结论和规则命中情况。
- 记录用户查看、追问、模拟、采纳或拒绝行为。
- 支持用户回看“当时为什么做出这个决定”。
- 支持比赛演示时展示完整的 Agent 执行轨迹。

#### 5.1.13 证据实验室

- 展示本次建议调用的 Pandadata 数据域、接口方法、查询区间和最新数据日期。
- 展示实际使用的 QuantSkills Skill/Agent，以及其运行状态和输出摘要。
- 展示 PandaAI 工作流节点图：数据下载、因子/规则构建、分析、回测、结果输出。
- 展示因子或策略的 IC、Rank IC、ICIR、衰减、分层收益、换手、最大回撤和夏普等研究指标。
- 对历史回测结果明确标注成交假设、交易成本、涨跌停处理、样本区间和“历史不代表未来”。
- 允许用户查看“支持证据”“反方证据”“证据不足”和“下一次复核条件”。

证据实验室默认折叠，避免理财小白被量化指标打扰；比赛演示时展开，用于证明系统不是聊天框套壳。

### 5.2 P1：增强完整度

- 失业、医疗支出、市场下跌等家庭财务压力测试。
- 周报和月报：目标进度、资产变化、风险变化和行动建议。
- 理财教练模式：通过提问帮助用户建立自己的判断框架。
- 伴侣或家庭共同目标管理。
- 将用户拒绝建议的原因纳入后续规划。
- 人工顾问复核入口。
- 证据实验室的完整因子/策略研究报告。
- PandaAI 工作流导出、审计和仿真运行。

### 5.3 P2：后续扩展

- 在满足牌照和合规要求后接入真实只读账户。
- 在独立确认和外部交易平台授权后提供下单跳转。
- 税务优化、保险保障分析和家庭资产负债表。
- 面向专业用户开放因子、回测和组合约束配置。
- 将用户资产画像与更多市场、基金和跨市场研究能力连接。
- 将经过本地验证的研究工作流发布为团队模板。

## 6. 信息架构与 UI

### 6.1 主导航

1. **首页**：回答“今天需要关注什么”。
2. **资产**：回答“钱在哪里、风险在哪里”。
3. **顾问**：处理目标规划、对话分析和建议解释。
4. **自选**：管理关注标的、观察理由和触发信号。
5. **我的**：管理目标、画像、提醒、授权和决策日志。

### 6.2 首页

- 总资产和财务状态摘要。
- 目标进度及预计达成时间。
- “今天需要你决定的事情”行动中心。
- 风险温度、应急金和配置偏离等核心指标。
- Agent 最近完成的分析任务。

首页不放大面积行情和涨跌榜，避免产品退化为行情软件。

### 6.3 资产页

- 按账户和资产类别切换。
- 展示资产配置、目标归属、集中度和流动性。
- 支持查看重复持仓和穿透后的行业暴露。
- 点击资产可进入持仓详情、历史建议和相关提醒。

### 6.4 顾问页

- 主体是对话，但不只输出文本。
- Agent 以目标卡、诊断卡、建议卡、比较表和模拟结果响应。
- 用户可展开“为什么”“数据来源”“反方观点”和“Agent 如何协作”。
- 涉及行动的回复必须通过结构化建议卡呈现。

### 6.5 自选页

- 列表默认展示与目标相关的状态，而不是只展示涨跌幅。
- 状态包括可观察、可分批增配、持有观察、风险升高和建议减配。
- 用户添加自选时可设置理由、目标、期限和提醒阈值。
- 用户可进入 Agent 分析页查看基本面、估值、组合影响和风险。

### 6.6 建议详情页

页面依次展示：

1. 建议动作与紧迫度。
2. 建议金额、比例和执行节奏。
3. 生成建议的三个主要理由。
4. 风险、反方证据和不确定性。
5. 模拟采纳后的组合变化。
6. 数据来源、更新时间和 Agent 执行记录。
7. 模拟采纳、询问 Agent、忽略或稍后处理。

### 6.7 提醒中心

- 按严重程度分组，不按发生时间简单堆叠。
- 风险类提醒优先于行情和资讯类提醒。
- 支持展开查看 Agent 正在执行或已经完成的分析。
- 提醒被确认后进入决策日志。

### 6.8 关键交互原则

- 先结论、后解释、再提供专业细节。
- 高风险动作增加二次确认和冷静期。
- 所有状态必须包含加载、数据过期、信息不足、分析失败和被风控拦截状态。
- 不用红绿涨跌刺激用户频繁交易；颜色主要表达风险等级和任务状态。
- 建议按钮使用“模拟采纳”，不使用“立即买入”或“立即卖出”。

## 7. Multi-Agent 架构

### 7.1 Supervisor / Planner Agent

- 理解用户请求或系统事件。
- 将任务拆成可执行步骤并生成依赖图。
- 决定哪些 Agent 可以并行执行。
- 管理状态、超时、重试、冲突和降级。
- 汇总候选结论，但不能绕过风险和合规审查。

### 7.2 画像 Agent

- 构建用户现金流、负债、目标、期限和风险画像。
- 判断主观风险偏好与客观风险承受能力是否矛盾。
- 识别损失厌恶、追涨杀跌等行为倾向。

### 7.3 数据 Agent

- 获取账户、行情、财报、指数、宏观和事件数据。
- 检查缺失值、异常值、复权方式、时间戳和数据新鲜度。
- 为后续结论提供可追溯的数据引用。
- 通过 `skill-pandadata-api` 将自然语言需求路由到明确的 `panda_data.get_*` 方法，先查接口契约再生成调用。
- 通过 `skill-pandadata-warehouse` 将重复使用的日线、分钟线、复权因子和研究因子增量缓存到 DuckDB + Parquet。
- 交易日统一由 `get_trade_cal` / `get_last_trade_date` 对齐，禁止使用自然日替代交易日。
- 每次取数生成 `DataSnapshot`，包含来源接口、参数、查询窗口、行数、最新数据日和校验状态。

### 7.4 研究 Agent

- 完成市场环境、行业、基金或标的研究。
- 生成支持与反对两侧证据。
- 可调用因子分析、检索、图表和报告工具。
- 单票研究优先路由到 `skill-a-share-stock-dossier`，输出公司画像、财务、资本运作、股东事件和资金面证据。
- 指数/行业研究优先路由到 `skill-index-valuation-rotation`，输出估值分位、估值温度和行业轮动摘要。
- 宏观研究优先路由到 `skill-macro-monitor`，要求先通过 `get_macro_detail` 确认指标代码，再进行同比、环比和发布日分析。
- 研究结果拆成“事实、计算值、推断、待验证假设”四类，禁止把推断伪装成接口事实。

### 7.5 组合 Agent

- 计算目标资产配置和组合偏离。
- 生成稳健、均衡和进取方案。
- 执行再平衡、情景模拟和目标成功率计算。
- 将买卖建议转化为组合比例和资金计划。
- 组合体检优先参考 `skill-portfolio-checkup` 的组合视角：集中度、行业偏离、事件敞口和基准偏离都按组合权重聚合。
- 当需要协方差和因子风险归因时，调用 `skill-risk-model` 的结构化风险模型能力；该 Skill 当前属于实验性资产，必须经过本地回归验证。
- 当需要生成约束下的权重时，调用 `skill-portfolio-optimize` 的候选权重计算；结果只能进入模拟比较，不能直接变成交易指令。
- 组合建议必须优先满足应急金、流动性、目标期限和资产上限，收益目标只能在约束满足后参与排序。

### 7.6 信号 Agent

- 监控自选和持仓的回撤、估值、事件和组合占比。
- 判断触发信号是否与用户目标相关。
- 输出增配、减配、持有或等待的候选建议。
- 为每个信号设置有效期和失效条件。
- 事件风险优先接入 `skill-event-risk-alert`，覆盖解禁、质押、减持、ST、业绩预告和审计意见等事件。
- 市场环境变化接入 `agent-market-regime-monitor`，将状态划分为延续、升级、降级或证据不足，而不是直接输出买卖结论。
- 热门资产风险接入 `agent-crowding-risk-monitor`，观察融资余额、龙虎榜热度、成交额和承接质量。
- 组合分散假设异常时接入 `agent-correlation-break-research`，判断相关性上升是否导致原有分散失效。
- 信号必须经过“标的层、组合层、目标层”三层过滤后，才允许升级为提醒或模拟建议。

### 7.7 风险 Agent

- 检查适当性、集中度、流动性和目标期限。
- 执行历史回撤、压力测试和极端情景分析。
- 对不适合用户的候选建议行使否决权。
- 使用事件、市场状态、拥挤度和相关性断裂四类外部证据做风险交叉验证。
- 对回测、因子和数据管线启用 `skill-numerical-leak-check`、生存者偏差检查和公司行为调整检查。
- 对组合清算问题可引入流动性压力测试，输出“需要多少交易日变现”“预计赎回缺口”和“冲击成本”。
- 风险输出必须区分“历史风险指标”“当前风险信号”和“情景假设”，不把三者混成预测。

### 7.8 合规审查 Agent

- 执行独立于 LLM 的硬规则检查。
- 拦截承诺收益、确定性预测、证据缺失和超出边界的表达。
- 检查建议是否包含风险、替代方案和有效期。
- 高风险或异常场景可要求人工复核。

### 7.9 解释与报告 Agent

- 将专业结论转化为普通用户可理解的语言。
- 生成建议摘要、证据链、反方观点和决策日志。
- 保持原始计算结果不被改写。
- 将官方 Skill 的报告包转换为建议卡：主结论、证据图、scorecard、决策矩阵、盯盘清单和交接卡。
- 研究报告中保留来源接口、查询窗口、数据日期和缺失说明，支持用户展开查看原文数据口径。
- 对所有历史回测指标统一添加“研究/教育用途，不代表未来表现”的说明。

## 8. 典型工作流

### 8.1 回撤触发卖出建议

1. 监控服务发现持仓达到回撤阈值。
2. Planner 创建回撤分析任务。
3. 数据 Agent 和画像 Agent 并行补全市场与用户信息。
4. 研究 Agent 判断市场、行业和标的层面的原因。
5. 组合 Agent 比较持有、减配和替代方案。
6. 金融引擎计算回撤、波动、目标成功率和压力测试结果。
7. 风险 Agent 检查用户适当性和组合约束。
8. 合规 Agent 检查表达、证据和建议边界。
9. 解释 Agent 生成建议卡。
10. 用户模拟采纳或拒绝，结果进入决策日志。

### 8.2 自选触发买入建议

1. 自选标的满足用户设置的观察条件。
2. 系统检查组合是否存在对应配置缺口。
3. 数据 Agent 验证行情、估值和事件数据的新鲜度。
4. 研究 Agent 提供支持和反方证据。
5. 组合 Agent 生成分批增配方案及替代资产。
6. 风险与合规双重审查。
7. 系统生成有期限的模拟增配建议。

如果数据不足、Agent 观点冲突或置信度低于阈值，流程降级为观察提醒或追问用户，不生成明确买卖建议。

### 8.3 PandaAI 证据工作流

对于需要向评委展示“Agent 如何完成复杂投研任务”的场景，使用 PandaAI 风格的工作流图作为证据载体：

1. Planner 将用户问题转换为工作流目标，例如“判断科技基金是否需要减配”。
2. 数据节点调用 Pandadata API 或本地仓库，输出带字段契约的行情、财务、事件和宏观数据。
3. 因子/规则节点生成回撤、估值、集中度、市场状态或事件风险信号。
4. 分析节点计算 IC、Rank IC、ICIR、衰减、分层收益、波动和回撤等研究指标。
5. 回测节点按明确的成交、成本、涨跌停和样本区间假设验证候选规则。
6. 风控节点执行适当性、流动性、集中度和目标约束。
7. 报告节点产出建议卡、证据图、决策矩阵和下一次观察清单。
8. 用户在前台查看“采纳、不操作、替代方案”的模拟比较。

PandaAI 手册中的模板模式、空白模式和 AI 对话模式对应三种内部任务入口：小白使用模板化诊断，系统自动构建空白工作流，进阶用户通过对话发起研究任务。工作流信号和仿真日志只作为研究验证材料，不作为实盘交易或投资建议。

## 9. 系统架构

### 9.1 客户端层

- 移动端优先的 Web PWA 或跨平台应用。
- 提供首页、资产、顾问、自选、提醒和个人中心。
- 图表只展示与目标和风险相关的信息。

### 9.2 API 与安全层

- 统一 API 网关和面向客户端的聚合接口。
- 身份认证、会话管理、限流、数据授权和字段脱敏。
- 所有敏感操作写入审计日志。

### 9.3 Agent 编排层

- 使用有状态工作流引擎表达任务依赖、并行节点和条件分支。
- 保存每个任务的输入、输出、工具调用和中间状态。
- 通过模型网关完成模型选择、超时、重试和成本控制。
- 增加 `SkillRegistry`：记录 Skill 名称、版本、来源仓库、验证级别、许可证、输入输出契约和本地冒烟测试结果。
- 增加 `SkillRouter`：根据任务类型、资产类别、数据域、资源成熟度和用户权限选择 Skill。
- 增加 `EvidencePackBuilder`：将各 Agent 的结构化结果拼装成可展示、可回放的证据包。
- 增加 `PandaWorkflowAdapter`：将内部任务图映射为 PandaAI 工作流节点图，支持导出、审计和仿真验证。

### 9.4 金融能力层

- 现金流与目标测算服务。
- 资产分类和穿透分析服务。
- 组合优化与再平衡服务。
- 风险指标、回撤、压力测试和回测服务。
- 提醒规则与适当性规则引擎。
- `PandadataAdapter`：封装 `panda_data` SDK、API 认证、方法契约、字段标准化和错误分类。
- `ResearchSkillRuntime`：运行单票尽调、指数估值、宏观监控、市场状态和拥挤度等研究 Skill。
- `BacktestValidationRuntime`：运行标准回测、过拟合检查、未来数据泄露检查和公司行为一致性检查。
- `SimulationAdapter`：将候选工作流接入模拟数据或 PandaAI 仿真环境，返回信号、模拟订单、成交、持仓和异常日志。

LLM 不直接承担数值计算，必须调用这些确定性服务。

### 9.5 数据层

- 关系数据库：用户、目标、账户、持仓、建议和决策日志。
- 时序数据存储：行情、净值和指标序列。
- 对象存储：研究报告、图表和导入文件。
- 检索索引：研究材料和产品知识。
- 审计存储：数据版本、模型输入输出、规则命中和操作轨迹。
- Pandadata 本地仓库：用增量水位线、Parquet 分区和 DuckDB 视图复用行情、复权因子和研究数据。
- Skill 产出仓库：保存 `report.html`、`scorecard.json`、`decision_matrix.csv`、`monitoring_checklist.csv`、`handoff_card.md` 等结构化产物。
- Evidence 关系：将每条建议关联到数据快照、Skill 运行、工作流节点、计算结果和风控规则。

### 9.6 运维与评测层

- 工作流追踪、模型调用监控和成本统计。
- 数据源健康检查和过期检测。
- Agent 成功率、冲突率、工具失败率和合规拦截率监控。
- 离线场景集、回归测试和人工抽检。
- Skill 冒烟测试：对每个接入 Skill 验证安装、触发、输入契约、输出 schema 和免责声明。
- 数据质量监控：检查交易日对齐、字段缺失、重复主键、覆盖窗口、API 与本地仓库抽样一致性。
- 研究质量监控：检查未来函数、生存者偏差、复权/公司行为、回测假设和样本外拆分。

## 10. 核心数据对象

- `UserProfile`：家庭、现金流、风险和行为画像。
- `Goal`：目标金额、时间、优先级和成功率。
- `Account`：账户类型、授权状态和更新时间。
- `Holding`：资产、数量、成本、分类和目标归属。
- `WatchItem`：自选标的、关注理由、期限和提醒规则。
- `AnalysisRun`：任务图、Agent 状态、工具调用和数据版本。
- `DataSnapshot`：Pandadata 方法、参数、字段、查询窗口、最新数据日、行数和质量状态。
- `SkillAsset`：Skill/Agent 名称、来源、版本、验证级别、许可证、输入输出 schema 和启用状态。
- `SkillRun`：一次 Skill 执行的输入、输出、运行日志、错误和质量门结果。
- `WorkflowRun`：内部工作流或 PandaAI 工作流的节点图、边、参数、状态和仿真日志。
- `Recommendation`：动作、比例、理由、证据、风险、有效期和状态。
- `Alert`：触发规则、严重级别、关联目标和处理状态。
- `Simulation`：方案输入和组合变化结果。
- `EvidencePack`：数据快照、研究报告、scorecard、反方证据、回测假设和风控结论。
- `DecisionLog`：用户的查看、追问、模拟、采纳或拒绝记录。

## 11. 异常与降级

- 数据源不可用：使用缓存并标记数据时间，无法满足新鲜度时停止建议。
- Pandadata 接口契约不匹配：停止本次 Skill，重新加载准确方法文档，不允许猜测参数或字段。
- 本地仓库校验失败：保留上一份有效分区，将新数据标为 `partial` 或 `failed`，不推进水位线。
- Skill 输出 schema 不匹配：隔离该版本并回退到已验证版本。
- PandaAI 工作流节点失败：保存节点级日志和中间结果，允许从失败节点恢复。
- 用户信息不足：生成追问任务，不猜测关键财务数据。
- Agent 结论冲突：展示冲突并追加验证节点，仍无法解决时降级为观察。
- 金融引擎失败：禁止使用 LLM 估算替代。
- 模型超时：保存工作流状态，允许恢复或使用能力较弱的备用模型。
- 风控拦截：明确告知建议未通过，并说明缺少的数据或违反的约束。
- 极端行情：提高提醒合并和冷静期，优先输出风险教育与组合影响。

## 12. 风险、合规与隐私

- 比赛原型不连接真实下单能力。
- 实际商业化前需根据目标市场确认投顾资质、数据授权和适当性要求。
- 建议不得包含保证收益、绝对化预测或诱导频繁交易的语言。
- 高风险建议必须经过风险和合规两个独立节点。
- 用户数据按最小必要原则采集，并支持导出与删除。
- 敏感数据加密存储，模型调用前进行脱敏。
- 外部研究资料视为不可信输入，防止提示注入影响 Agent 工作流。
- 决策日志可追溯，但不得泄露其他用户或内部敏感信息。

## 13. 评测与测试

### 13.1 产品指标

- 完成建档并生成首个目标方案的比例。
- 用户能否正确理解建议原因和主要风险。
- 建议被模拟、追问、采纳和拒绝的比例。
- 提醒有效率和通知关闭率。
- 目标计划的持续跟踪率。

### 13.2 Agent 指标

- 复杂任务完成率。
- 工具调用成功率和数据引用覆盖率。
- Agent 冲突率和自动解决率。
- 建议中事实、计算与证据的一致性。
- 合规规则命中率和漏检率。
- 单次任务耗时和模型成本。
- Pandadata 方法路由准确率和字段契约一致率。
- Skill 输出 schema 通过率、冒烟测试通过率和版本回退次数。
- EvidencePack 完整率：数据日期、来源接口、计算公式、反方证据、回测假设和风险结论是否齐全。

### 13.3 核心测试场景

1. 用户风险偏好与财务承受能力矛盾。
2. 单一行业持仓超过 30%。
3. 自选标的下跌但组合没有配置缺口。
4. 持仓回撤超过阈值但基本面和目标没有变化。
5. 买房时间提前导致流动性需求上升。
6. 账户数据缺失或行情数据过期。
7. 研究材料包含提示注入内容。
8. 多个 Agent 给出相反结论。
9. 金融计算服务不可用。
10. 用户要求“保证收益”的买入建议。
11. Data Agent 选择了错误的 Pandadata 方法或使用不存在的字段。
12. 本地仓库与在线 API 抽样结果不一致。
13. 回测结果存在未来数据泄露、生存者偏差或复权错误。
14. QuantSkill 的验证级别、状态或版本发生变化。
15. PandaAI 工作流中断后能否从节点级状态恢复。
16. 仿真结果与建议卡引用的指标不一致。

## 14. MVP 范围

3. 创建买房目标并生成三档配置方案。

## 15. 比赛演示流程

建议使用一名 28 岁、计划三年后买房的上班族作为演示用户。

1. 用户用一句话说出买房目标，Agent 通过三到五轮对话完成建档。
2. 系统导入模拟账户，发现科技基金占比过高、应急金不足。
3. 用户添加宽基基金到自选并设置回撤提醒。
4. 模拟市场事件触发科技基金回撤和集中度警报。
5. Data Agent 通过 `skill-pandadata-api` 展示实际数据路由，并从本地仓库复用已缓存数据。
6. 市场状态、拥挤风险、事件风险和单票/基金研究节点并行运行。
7. 页面实时展示 Planner、数据、研究、组合、风险和合规 Agent 的工作状态。
8. 系统生成分批减配建议，并展示证据、反方观点和失效条件。
9. 用户打开 Evidence Lab，查看 PandaAI 风格工作流图、回测假设、风险 scorecard 和 Skill 输出。
10. 用户点击模拟采纳，目标成功率提高、组合波动下降。
11. 用户询问“如果我不卖会怎样”，系统比较三种方案。
12. 最后展示决策日志、PandaAI 仿真日志和完整 Agent 执行轨迹。

该流程可以同时体现真实用户价值、AI 原生交互、多 Agent 复杂任务、工具调用、金融专业性和合规边界。

## 16. 验收标准

- 理财小白无需理解专业术语也能完成核心流程。
- 每项建议均能追溯到用户目标、数据、计算和 Agent 结论。
- 买入或卖出建议不会由单一行情信号触发。
- 风险或合规节点可以阻断不适当建议。
- 所有数值结果均来自确定性金融服务。
- 数据缺失或过期时系统不会编造结果。
- 用户可以比较方案、模拟采纳并保留最终决定权。
- 演示能够完整展示一次 Multi-Agent 任务从触发到反馈的闭环。
- 每个 Pandadata 调用均能追溯到明确的方法契约、参数和数据日期。
- 接入的 QuantSkill 均有验证级别、版本、许可证和本地冒烟测试记录。
- Evidence Lab 可以回放 PandaAI 工作流、Skill 产物、计算结果和风控节点。

| 级别 | 产品策略 |
| --- | --- |
| L3 Verified + active | 可作为核心研究/监控节点，但仍需本地数据和输出回归测试 |
| L2 Runnable + stable | 可进入 MVP 运行时，必须锁定版本并通过冒烟测试 |
| L2 Runnable + draft | 可用于内部研究，不直接驱动用户建议 |
| L1 Listed 或未验证 | 只作为方法参考或实验模块，不能成为高风险建议的唯一证据 |

验证级别不等同于投顾资质、收益保证或监管认可。任何 Skill 的输出仍必须经过本产品的用户适当性、组合约束、风险和合规节点。

### 17.2 Pandadata API 方法地图

产品不直接把 218 个方法全部暴露给 Agent，而是按业务能力建立白名单：

| 能力 | 首选方法 |
| --- | --- |
| 交易日与日期对齐 | `get_trade_cal`、`get_prev_trade_date`、`get_last_trade_date` |
| A 股行情与复权 | `get_stock_daily`、`get_stock_rt_daily`、`get_stock_daily_pre`、`get_stock_daily_post`、`get_adj_factor` |
| 基金和 ETF | `get_fund_detail`、`get_fund_daily`、`get_fund_daily_pre`、`get_fund_daily_post`、ETF 申赎相关方法 |
| 指数与估值 | `get_index_daily`、`get_index_weights`、`get_index_indicator` |
| 财务与业绩 | `get_fina_reports`、`get_fina_performance`、`get_fina_forecast`、`get_audit_opinion` |
| 股东与事件风险 | `get_restricted_list`、`get_stock_pledge`、`get_stock_shareholder_change`、`get_stock_status_change` |
| 资金与交易行为 | `get_margin`、`get_hsgt_hold`、`get_lhb_list/detail`、`get_block_trade` |
| 宏观与经济日历 | `get_macro_detail`、宏观分类方法、`get_macro_cal` |
| 港美股 | `get_hk_daily`、`get_us_daily`、公司事件、一致预期和财务方法 |
| 期权和波动率 | `get_option_daily`、`get_option_implied_volatility`、`get_option_underlying_volatility` |
| 量化研究 | `get_factor`、`get_adj_factor` |

每次调用必须先由 `skill-pandadata-api` 读取精确参数和字段契约；上表只负责业务路由，不替代接口文档。

### 17.3 PandaAI 平台能力映射

| PandaAI 能力 | Money Whisperer 中的用途 |
| --- | --- |
| 自然语言生成策略 | 将用户问题转换为研究工作流草案，由 Planner 和用户共同确认 |
| 官方模板 / 模板模式 | 提供回撤分析、组合体检、宽基定投观察等标准工作流 |
| 空白工作流 | 供 Agent 动态生成多节点研究任务 |
| Python / 公式节点 | 实现确定性指标、因子、规则和数据转换 |
| 参数可视化调整 | 支持比较回撤阈值、目标期限、配置上限和投入节奏 |
| IC/ICIR 与收益风险报告 | 作为证据实验室的研究验证指标 |
| 超级图表 | 展示因子、策略和条件触发位置，不作为买卖按钮 |

### 17.4 Skill 运行契约

每个 Skill 必须注册以下元数据：

- `asset_name`、`repository`、`commit_sha` 或版本。
- Registry 验证级别、状态和最近一次核对日期。
- 许可证及允许的复用方式。
- 输入 schema、输出 schema、所需数据源和凭证。
- 是否允许进入用户建议链，或仅允许用于研究/验证。
- 超时、重试、缓存、降级和回退版本。
- 冒烟测试、样例输出和免责声明检查结果。

### 17.5 资源安全边界

- QuantSkills 是研究资产库，不等同于监管认证的投顾模型。
- PandaAI 工作流、超级图表和仿真结果用于研究与验证，不构成真实交易指令。
- 比赛原型不自动下单、不接券商交易接口，所有买入/卖出动作都停留在模拟采纳和用户决策记录。
- GPL 等许可证需要在代码复用前进行合规审查；不满足商业化要求时只复用方法论和接口思想。
- 社区 Skill 的阈值和默认规则不能直接当作本产品的用户适当性规则。
- 所有外部 Skill 输出视为不可信输入，必须经过 schema 校验、数值检查和提示注入防护。
