# Money Whisperer 多 Agent 目标理财管家

## 产品语言

中文（简体）。所有产品界面、Agent 对话、建议卡、证据实验室展示、提醒文案与决策日志均使用中文。技术标识（如 Skill、Agent、Pandadata、PandaAI）保留原文以尊重官方资源命名。

## 屏幕类型

Desktop primary, responsive to mobile

理由：比赛演示、Evidence Lab 与多 Agent 执行轨迹需要充分利用桌面大屏，同时兼容用户在手机端查看目标、资产、提醒和建议卡。

## 要构建什么

Money Whisperer 是一款 AI 原生、多 Agent 协作的目标理财管家原型。用户用一句话描述生活目标（例如"三年后买房"），系统调度 Planner / 画像 / 数据 / 研究 / 组合 / 信号 / 风险 / 合规 / 解释 等 Agent，围绕真实财务状况生成"增配 / 减配 / 持有 / 观察"结构化建议，并全程展示证据链、反方观点、模拟采纳与失效条件。原型专注展示**双赛道能力**：AI 原生理财顾问体验 + 多 Agent 实战工作流；不连接真实交易，只支持模拟采纳。

产品前台强调"简单、可信、可行动"，后台通过 Evidence Lab（证据实验室）向进阶用户/评委展示 Pandadata 数据路由、QuantSkills 运行、PandaAI 工作流与仿真日志。

## 目标用户

主要人群：22-40 岁、收入相对稳定的上班族。

- 有一定存款或基金 / 股票资产，但缺乏系统理财知识；
- 正在为买房、应急金、教育、养老等目标积累资金；
- 资产分散在银行卡、基金平台、证券账户中；
- 容易被短期行情和社交媒体观点影响；
- 不知道自己真正能承担多少风险、如何配置、看不懂投研结论。

MVP 演示锚定角色：**28 岁计划三年后买房的上班族**（对应比赛演示流程）。
次要观众：比赛评委 / 进阶用户（通过 Evidence Lab 查看数据、Skill、工作流、风控证据）。

## 产品应用/门户

单一消费者端 App（Money Whisperer 用户端）。Evidence Lab 是同一 App 内的进阶视图（默认折叠，可展开），不作为独立门户。原型阶段无独立后台管理端、无顾问端。

## 功能与页面

### 主导航（5 个 Tab）

1. **首页** — 今日关注、总资产摘要、目标进度、"今天要决定的事"、风险温度、Agent 最近任务
2. **资产** — 账户/类别切换、目标归属、集中度、流动性、穿透式行业暴露、重复持仓识别
3. **顾问** — 对话式建档 / 追问 / 目标规划 / 建议解释；Agent 以结构化卡片响应（目标卡、诊断卡、建议卡、比较表、模拟结果）
4. **持仓观测** — 观察标的、关注理由、期限、回撤阈值、Agent 结论；状态基于目标而非涨跌
5. **我的** — 目标、画像、提醒、授权、决策日志

### 关键子页面

- 对话式建档（多轮采集 + 风险偏好情景问答 + 矛盾检测）
- 资产聚合导入（模拟账户 / 手工录入 / CSV）
- 财务健康诊断（结余率、负债率、应急金覆盖、集中度；输出三项最重要问题）
- 目标规划（稳健 / 均衡 / 进取 三档方案 + 优先级冲突分析）
- 建议详情页（动作、比例、节奏、三大理由、风险与反方、模拟采纳后组合变化、数据来源与 Agent 执行记录）
- 模拟采纳页（对比采纳/不操作/替代方案 三种结果）
- 提醒中心（按严重程度分组：信息 / 关注 / 重要 / 紧急）
- 决策日志（可回放"当时为什么这么决定"）
- **Evidence Lab（证据实验室）** — 展示 Pandadata 数据域与接口、QuantSkills 运行状态、因子/回测指标、支持/反方证据、下一次复核条件；默认折叠，进阶展开
- **Multi-Agent 执行轨迹面板** — 实时展示 Planner / 画像 / 数据 / 研究 / 组合 / 信号 / 风险 / 合规 / 解释 各 Agent 的工作状态

### 已确认的原型呈现决策

- MVP 主闭环：买房目标 → 集中度/回撤触发 → Multi-Agent 减配建议 → 模拟采纳。
- 首页提供“演示模式”开关，一键切换普通用户视图与评委视图。
- 主界面用剧场式 Agent 卡片流，Evidence Lab 使用任务依赖图展示并行与串行协作。
- 演示画像：28 岁上班族，月入 2 万，存款 15 万，科技基金占比过高，应急金不足，三年后买房。
- 图表分层：主界面展示配置、回撤和目标进度等中等深度图表；Evidence Lab 展示专业研究指标与图表。
- 访客直进：不强制登录，默认以匿名账号进入；“我的”中的录入存到当前匿名账号底下，服务端按账号隔离；可选后续绑定邮箱账号以跨设备同步。

### 核心 Agent 工作流（P0 演示闭环）

1. 用户对话建档 → 生成画像
2. 导入模拟资产 → 财务健康诊断
3. 创建买房目标 → 生成三档配置方案
4. 添加持仓观测 + 设置回撤阈值
5. 触发集中度 / 回撤事件
6. Multi-Agent 并行研究、模拟、风控、解释
7. 生成结构化增配 / 减配建议（带证据、反方、有效期）
8. 用户模拟采纳 → 目标成功率变化可视化
9. 决策写入日志
10. 展开 Evidence Lab 查看 Pandadata 接口、QuantSkill、PandaAI 工作流、风控拦截

### 数据与外部资源（原型使用模拟数据，接口结构保持一致）

- Pandadata API（模拟响应）：行情、财务、指数、宏观、事件
- QuantSkills：skill-pandadata-api、skill-pandadata-warehouse、skill-a-share-stock-dossier、skill-event-risk-alert、skill-index-valuation-rotation、skill-macro-monitor、agent-market-regime-monitor、agent-crowding-risk-monitor、agent-correlation-break-research 等
- PandaAI：工作流、超级图表、仿真模块（模拟展示）

### 关键设计原则

- 目标优先、组合优先、风险优先、证据优先、行动可逆、人机共决、规则兜底
- 首页不放大面积行情 / 涨跌榜，避免退化为行情软件
- 建议按钮统一为"**模拟采纳**"，不使用"立即买入 / 卖出"
- 颜色主要表达风险等级和任务状态，而非涨跌红绿
- 所有状态必须覆盖：加载 / 数据过期 / 信息不足 / 分析失败 / 被风控拦截
