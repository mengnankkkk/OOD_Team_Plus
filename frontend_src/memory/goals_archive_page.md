用户希望账号下拉里除了「个人财务档案」再新增一个「个人目标档案」入口，点击后进入独立页面「我的个人目标档案」。

**Why:** 用户明确要求：点击游客后，除了有"个人财务档案"，增加"个人目标档案"。"个人目标档案"点击后显示"我的个人目标档案"，可以增删改自己创立的目标，没有目标数量限制，目标有"目标类型"、"资产缺口"等（自行补充）。

**How to apply:**
- 顶栏账号下拉里在「个人财务档案」下方新增「个人目标档案」，跳转 /goals。
- 「个人财务档案」页面底部也加了一个「打开个人目标档案」按钮，方便两个档案互相跳转。
- `GoalsPage` 展示概览四联（目标总数、目标金额合计、已备金额合计、资产缺口合计），下方目标卡片双列布局。
- 每张目标卡片包含：目标类型徽标（购房/应急金/教育金/养老/自定义）、优先级 P1-P5、目标名称、类型副标、进度条与百分比、四个关键指标（资产缺口、目标日期与剩余月份、月度供款与自动估算、成功概率占位）、创建/更新时间、编辑与删除按钮。
- 新建/编辑对话框收集：目标名称、目标类型、优先级、目标金额、已备金额、目标日期、月度供款（可留空 → 自动按 gap ÷ 剩余月份估算）；实时预览「资产缺口」。
- 删除走 AlertDialog 二次确认，删除时会解绑资产但保留历史建议与决策日志。
- 所有操作走 `goalService`：`listGoals` / `createGoal` / `updateGoal` / `deleteGoal`，均基于 `auth.uid()` 与 RLS，无数量上限（RLS 只按 user_id 限制）。
- `useUserGoals` 现在附带 `useUserGoalsInvalidator`，供 CRUD 之后失效缓存。
- 目标类别 (`goals.category`) 数据库 CHECK 约束仅接受：house / emergency / education / retirement / custom；新增其他类型前需要先改 DDL。
