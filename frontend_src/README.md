# Money Whisperer · 财语

## 对话式建档与风险画像

演示里我们让"林然"三年买房这条主线跑通，但用户画像还是一张写死的样板。下一步要让每个真实用户都能坐到这张桌子前：

- 让**通用智能体**承担"顾问 Agent"的角色 —— 用户用一句话说出目标后，它会依次追问收入、支出、负债、家庭责任、既有资产，遇到"主观激进但客观承受能力不足"这类矛盾时主动回敲。
- 用**账户体系**给每位用户开一个独立的档案抽屉，包含目标、风险等级、行为倾向、隐私授权范围与删除入口，用户之间彼此隔离，除了本人和自己授权的顾问模式，任何人都读不到。
- 用**真实存储**保存每一轮对话摘要、画像版本和"我为什么这么答"的原因，日后每次生成建议都能追溯到当时那份画像。
- 每一条顾问回复下方都挂一条**思维链** —— 展开后可以看到这次 trace 里每一个 span 的名字、调用的工具、传入的参数、返回的 JSON 与最后归结的 loop 回复；不想看时一键折叠回去。
- 顾问页采用**双栏会话布局**：左侧的“新建会话”与“历史会话”列表方便在不同话题之间切换或重启对话，右侧为当前会话的主对话区，不同会话之间上下文不互相污染。

用户拿到的是：不再需要填冷冰冰的问卷；换设备登录后画像还在，愿意更新时随时可以修订；想较真时能开思维链开到链路层面。

## 资产聚合与财务体检

首页那张"科技基金 62%、应急金 1.8 个月"的红灯目前是硬编码的，接下来我们让它真正来自用户自己的账户：

- 提供**手工录入 + CSV 导入**两条通道，覆盖现金、存款、货币基金、债券基金、权益基金、股票；文件解析走**通用智能体**，识别列名、日期与币种，遇到脏数据会请用户逐条确认，而不是猜。
- 用**真实存储**把持仓、成本、目标归属、更新时间都存下来，并按用户切分 —— 每个人只看到自己的账本，页面加载时按账户分批读取，避免一次性把所有历史流水拉到前端。
- 财务健康四联仪表（应急金覆盖、储蓄率、集中度、最大回撤）改为按用户当下的资产实时计算；穿透后的行业暴露、重复持仓、流动性缺口也用同一份数据算出，不再各说各话。

用户拿到的是：把散在各个 App 的账本合到一处，一眼看到"钱在哪里、风险在哪里"，而不用自己拿计算器加总。

## 资产 · 语义层

在资产板块下新增一个「语义层」子页面组，用来给每个物理表建立业务侧的语义映射：

- 顶栏新增「**语义层**」下拉，三个入口：**领域管理 / 表管理 / 外键管理**；「字段管理」作为「表管理」的子页存在，需要先选中具体的表再进入。
- **领域管理**：按业务领域分组（交易 / 用户 / 营销 …），可增删改，可切换是否对下游可见。
- **表管理**：为物理表补充语义名、语义描述、所属领域、schema、同步状态、可见性；每行可以直接跳进它的「字段管理」子页。
- **字段管理**（表管理子页）：维护该表下每个字段的物理定义（类型 / nullable / 主键 / 默认值）、语义名、业务类型（id / user_id / money / status …）、示例值与可见性。
- **外键管理**：手工或 AI 推断录入两张表之间的逻辑关联，记录关系类型（many_to_one 等）、来源与置信度，可随时切换是否生效。

数据同样按用户隔离，领域 / 表 / 字段 / 外键均持久化，支持无限新增与销毁。

## 多 Agent 建议引擎（核心）

演示里那条 Agent 剧场卡片流讲清楚了"这条建议是怎么做出来的"，但六个 Agent 目前只是状态图。真要把它变成能出建议的引擎，我们做两件事：

- 用**通用智能体**承载 Planner / 画像 / 数据 / 研究 / 组合 / 信号 / 风险 / 合规 / 解释 九个角色，按任务图并行或串行协作。Planner 拆任务、分派、处理冲突和重试；风险与合规两个节点保留独立的否决权，LLM 无法绕过。
- 用**后台函数**把确定性的金融计算封装成工具：目标缺口、结余率、集中度、再平衡权重、历史回撤、压力测试、组合成功率 —— 数值一律由函数返回，Agent 只负责调用与解释，不允许自己算。

围绕演示的减配主闭环，我们同时打磨三条日常会触发的建议流水线：**回撤触发的减配**、**目标资金缺口触发的增配**、**持仓观测阈值触发的观察或分批增配**。每一次输出的建议都会强制携带绑定目标、动因、证据链、有效期、失效条件五个字段 —— 缺一个，就不允许出现在用户面前。

用户拿到的是：真正来自自己财务状况的"今天要决定的事情"，而不是通用文案；点开每张建议卡，都能看到它为什么被生成，以及什么时候会作废。

## 证据实验室与演示模式

演示里的"评委视图"目前只有一个静态徽标，下一步我们让它变成可展开的第二层：

- 顶栏保留**演示模式开关**：普通视图给理财小白，评委视图揭开每张卡的批注角标、Pandadata 数据路由、Agent 运行耗时和风控拦截原因。
- 顶栏新增**“历史记录”多栏下拉**：点一下就能看到 “决策日志” 与 “Evidence Lab” 两张回溯入口卡片，各自带图标、说明与归属提示，直接跳到对应页面。
- 账号头像下拉新增**“个人目标档案”**：不限数量地管理自己的购房、应急金、教育金、养老与自定义目标，每个目标展示目标金额、已备金额、资产缺口、目标日期、建议月度供款、完成度、优先级，可随时新增、修改与删除。
- **真实存储**保留每一次建议背后的证据包：调用了哪些数据接口、哪些查询、最新数据日期、支持与反方证据、回测假设。用户或评委可以在建议详情页里一键跳到 Evidence Lab，落到具体那三个节点。
- Evidence Lab 里的任务依赖图和 Pandadata / QuantSkills / PandaAI 面板改为读实时运行记录，而不是示意图；同一份数据同时给普通视图当"眉批"、给评委视图当"底稿"。

用户拿到的是：想简单的时候界面就是简单的；想较真的时候，每一条建议都能沿着红线走回它出生的那次查询。

## 提醒中心与决策日志

演示里"今天要决定的事情"是一张，真实世界里会有很多张。为了不让它退化成消息轰炸，我们做：

- **实时同步**：账户资产、市场触发条件、Agent 判断结论一旦更新，建议卡与提醒会主动推到用户当前打开的页面上，不需要手动刷新。
- **提醒分层**：按信息 / 关注 / 重要 / 紧急四级分组，相似提醒自动合并；用户可以选"仅重要"、"每日汇总"或"完全静默"。
- **决策日志**用**真实存储**记下用户每一次查看、追问、模拟采纳、拒绝以及当时的原因；一个月后回看"我当时为什么这么决定"，画面和当时几乎一样。

用户拿到的是：不再被无差别的行情推送打扰；每一次做过的决定都可以复盘。

## 模拟采纳与可撤销行动

演示里"模拟采纳"是一枚朱砂章，真实版本里它要能被落下、也能被撤回：

- 用**真实存储**记录每一枚"已模拟采纳"章：调仓前后的资产结构、预期波动、目标成功率变化、失效日期。
- 用**后台函数**在采纳、撤销、比较三种动作之间保持数据一致 —— 撤销后回滚整块状态，不留半截数据。
- 演示阶段坚决不接真实下单，所有印章上都会带"仅供研究"的水印；未来接入前会与您和合规方单独确认。

用户拿到的是：可以放心地在演示账本上试各种方案，随时反悔，也随时对比"如果我不采纳会怎样"。

## 访客直进与隐私边界

这些能力叠在一起就要求一件事 —— 每个人只能看到自己的世界，但没有登录的用户也能直接开始：

- 默认打开页面就进入**游客模式** —— 后台自动为该浏览器开一个属于你的临时账号，你在“我的”里录入的基本信息、在其他页面添加的资产、产生的建议与提醒都存在这个账号下，下次打开仍然看得到。
- 想跨设备同步或长期保留时，在“我的”里选择**绑定邮箱账号**，临时账号就会汇入你的个人账号。
- 不论是临时账号还是已绑定账号，数据访问在服务端强制按用户切分，前端不做安全兜底 —— 即使有人绕过界面，也拿不到别人的账本。
- 资产表、决策日志这类列表加载时按分页取，避免一次性把几年的历史全部拉到手机上。

用户拿到的是：不需要先注册就能看到完整体验；想让数据跟着你到其他设备时，再一键把这份账本绑到你的邮箱。

## How it all connects

底层是一台承担九个 Agent 协作与数据存取的**中枢服务**，两侧是普通用户看到的界面和评委看到的实验室，中间靠登录和账户体系把每个人的世界隔开。

```
+---------------------------------------------------------------+
|                        用户使用的界面                          |
|   首页 · 资产 · 顾问 · 持仓观测 · 建议详情 · 决策日志              |
+---------------------------------+-----------------------------+
                                  |  游客或登录账号，按用户切分数据
                                  v
+---------------------------------------------------------------+
|                          中枢服务                              |
|  - 九个 Agent 协作：规划 / 画像 / 数据 / 研究 / 组合 /          |
|                     信号 / 风险 / 合规 / 解释                  |
|  - 金融计算与压力测试由确定性函数完成，Agent 只调用不自算       |
|  - 每一次建议、模拟、撤销、拒绝都记进决策日志                   |
|  - 账户变化与市场触发实时推送到用户当前的页面                   |
+----------------+-----------------------------+----------------+
                 ^                             ^
                 |  切换演示模式                | 用同一份运行记录
                 |                             |
+----------------+---------------+   +---------+---------------+
|         普通用户视图            |   |    Evidence Lab 视图    |
|  目标进度 · 建议卡 · 提醒中心    |   |  数据路由 · Agent DAG · |
|                                |   |  证据链 · 回测假设       |
+--------------------------------+   +-------------------------+
```

- **用户使用的界面** —— 五个主导航加建议详情与决策日志，是林然（以及未来每一位真实用户）每天真正打开的地方。
- **中枢服务** —— 九个 Agent 的协作剧场、确定性金融计算、账户与决策存储、市场触发的实时推送都在这里。用户之间彼此隔离，高频列表分页加载。
- **普通用户视图 / Evidence Lab 视图** —— 同一份运行记录，两种呈现：想简单看结论就用左边，想较真每一条证据就切到右边。
